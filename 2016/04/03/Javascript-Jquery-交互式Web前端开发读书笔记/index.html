    1. 事件冒泡：事件是从最具体的节点开始向外传播到最宽泛的节点。这是事件流的默认类型，被绝大多数浏览器所支持。
    2. 事件捕获：事件从最宽泛的节点开始向内传播到最具体的节点。这种方式在IE8和更早版本的lE中不被支持。

    Web存储API：在浏览器中存储数据

    ​ HTML5引入了存储对象。存储对象分为两种，localStorage和sessionStorage。它们拥有相同的方法和属性。主要区别在于数据能保存多久，以及是否所有选项卡都能访问已经存储的数据。

    存储 本地存储 会话存储
    关闭窗口/选项卡后数据是否依然存在？ 是 否
    已经打开的所有窗口是否都可以访问数据？ 是 否

    ​ 通常，浏览器会为每个域的存储对象分配5MB的数据空间。如果网站尝试存储超过5MB的数据，浏览器就会询问用户是否允许该网站存储更多信息（永远都别指望用户会同意）。

    ​ 数据会作为存储对象的属性存储起来（使用键值对）。键值对中的值是字符串。为了保护网站存储在这些存储对象中的信息，浏览器采取了同源策略，也就是说只有位于相同域中的其他页面才能访问这些数据。

    ​ http:// www. google.com: 80

    URL的4部分都必须匹配：

    1. 协议：协议必须匹配。如果在以http其实的页面上保存了数据，那么此数据就无法通过https来访问。
    2. 子域名：子域名必须匹配。例如，maps.google.com 无法访问www.google.com上存储的数据。
    3. 域名：域名必须匹配。例如，google.com 不能访问facebook.com上存储的数据。
    4. 端口号: 端口号必须匹配。Web服务器可以使用各种端口号。URL中通常不会指定端口号，这是因为网站为其Web页面使用默认的80端口，但端口号是有可能变化的。
